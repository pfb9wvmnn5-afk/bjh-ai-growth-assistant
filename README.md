# 白金汉宫 AI 增长参谋

这是一个纯静态网页工具，适合部署到 GitHub Pages。

## 功能

- 内容搜索：用 AI 搜索白金汉宫内置知识和用户新增知识，并基于知识库整理回答，不编造。
- 选题策划：根据平台、目标、宴会类型、门店和今日素材生成选题。
- 内容创作：生成金辉口播、短视频脚本、直播脚本、朋友圈文案等。
- 销售回复：粘贴客户微信原话，生成可复制回复。
- 知识库更新：新增知识只需填写正文，可先点击“AI 整理预览”自动生成标题、分类、场景、关键词和整理后的正文，确认无误后再保存；也支持搜索、编辑、删除、导入、导出自定义知识。
- 知识库清理：搜索后可勾选知识，批量停用内置知识、恢复停用知识、删除自定义知识。
- 配置：用户自行填写 API Key、Base URL 和模型名。
- Word 导出：知识问答、选题策划、内容创作、销售回复结果都可以导出为 Word 可打开的 `.doc` 文件。

## 使用说明

打开 `index.html` 后，进入“配置”填写自己的模型信息。

网页自带一份内置白金汉宫基础知识库，所有人获得链接后都能直接使用这些基础知识。

所有配置和个人新增知识都保存在当前浏览器 `localStorage` 中，不会上传到云端。换设备或换浏览器前，请先在“知识库”页面导出 JSON，再到新设备导入。

新增知识推荐流程：先把正文粘贴进去，点击 `AI 整理预览`，确认标题、分类、场景、关键词和正文都没问题，再点击 `确认保存知识`。保存成功后，这条知识会立刻出现在右侧知识列表，并正式参与后续 AI 搜索和生成。

所有生成模块只保留 `AI 生成`。使用前需要先进入“配置”，选择模型方案并填写自己的 API Key。

内容搜索已经替代原来的知识问答：只需要输入搜索内容，不需要选择搜索范围和搜索方式。AI 会根据当前启用知识库整理结果，但不能扩展知识库没有的信息。

生成结果会自动分成清晰卡片，方便员工在手机上快速看懂和复制。内容搜索结果不再做关键词彩色高亮，只保留清楚的板块和段落。默认回答会保持简洁：问什么答什么，不主动扩展销售话术、短视频、岗位动作或复盘方案；只有明确要求详细、依据、方案、话术拆解时才展开。回答风格已调整为真人业务助手：简单问题短答，销售回复默认像微信可直接发送，内容创作更像自然口播。销售回复支持“大白话口语版”，选择“给3个版本”时会输出亲切自然、大白话、推进成交三版。

事实边界：AI 生成涉及宴会厅名称、门店地址、套餐、价格、优惠、活动、客户案例、数据和政策时，都必须来自知识库。知识库没有就提示“当前知识库暂未记录”，不能编造。

本轮已调整为 AI-only：页面不再提供本地生成按钮。

知识列表搜索框用于管理知识库，搜索命中的标题、分类、场景、关键词和正文片段会高亮显示。右侧列表只显示摘要；点“打开编辑/打开修订”后，左侧会打开这条知识的完整正文，可以在完整内容里手动删除或修改词语、句子、段落后再保存。

知识库里的内置知识不能直接删除，但可以停用。打开内置知识修订后，保存会生成一条自定义修订版，并自动停用原来的内置条目，避免旧内容继续参与 AI 提示词和知识搜索匹配；勾选“显示已停用”后可以恢复。自己新增的自定义知识可以直接删除或批量删除。

配置页里选择模型方案后，Base URL 和模型名会自动填写。模型方案按平台统一显示，例如 OpenAI、DeepSeek、通义千问、Kimi、自定义。模型名可以手动修改，方便某个模型版本下线时改成新模型名。

每次生成都会自动保存到对应模块的历史版本里，历史版本可以查看或删除。每个模块也支持填写“调整要求”，再重新生成新版本。

如果后续需要“公司统一知识库”，让所有员工自动看到最新知识，需要升级成云端知识库版本。

## 部署

把本目录中的文件上传到 GitHub 仓库，然后在 Settings -> Pages 中启用 GitHub Pages。

如果 `index.html` 在仓库根目录，访问地址通常是：

```text
https://用户名.github.io/仓库名/
```

如果本目录作为子文件夹上传，访问地址通常是：

```text
https://用户名.github.io/仓库名/bjh-ai-growth-assistant/
```

## 安全边界

不要把统一 API Key 写进代码。每位使用者在自己的浏览器里填写自己的 API Key。
